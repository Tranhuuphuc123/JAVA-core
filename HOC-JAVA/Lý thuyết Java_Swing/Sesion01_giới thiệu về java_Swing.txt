***********************TIỀM HIỂU VỀ JAVA SWING****************************
>>> Giới thiệu:
 - Java Swing là một phần của Java Foundation Classes (JFC) được sử dụng để tạo các ứng dụng window-based. Nó được xây dựng trên API AWT
  (Abstract Windowing Toolkit) và được viết hoàn toàn bằng Java.

  - Không giống như AWT, Java Swing cung cấp các thành phần không phụ thuộc vào nền tảng và nhẹ hơn.

  - Gói javax.swing cung cấp các lớp cho java swing API như JButton, JTextField, JTextArea, JRadioButton, JCheckbox, JMenu, JColorChooser, v.v.
  

>>>> JFC là gì?
 - Java Foundation Class (JFC) là một bộ các thành phần GUI đơn giản hóa sự phát triển của các ứng dụng desktop.

>>>> Sự khác nhau giữa lib AWT và Swing
  
        No.	                 Java AWT                                             	   Java Swing
        
        1)      Các thành phần AWT là phụ thuộc nền tảng.       	     Các thành phần Java swing là độc lập nền tảng.
        2)	    Các thành phần AWT là nặng.         	                 Các thành phần Swing là nhẹ.
        3)	    AWT không hỗ trợ plugin.	                             Swing có hỗ trợ plugin.
        4)	    AWT cung cấp ít thành phần hơn Swing.	                 Swing cung cấp nhiều thành phần mạnh mẽ hơn AWT như tables, 
                                                                         lists, scrollpanes, colorchooser, tabbedpane, vv.

        5)	    AWT không tuân theo MVC(Model View Controller)           Swing tuân theo mô hình MVC.
              trong đó model biểu diễn data, view biểu diễn hiển thị 
              và controller biểu diễn các action để kết nối model với
               view.

 -                                          Model

                ++++ trong đóa:
                 > Model: có chức năng quản lý và xử lý dữ liệu.(vd như JDBC tg tac java với SQL,  Method chính thực thi chương trình)
                    + Là bộ phận có nhiệm vụ quản lý dữ liệu của ứng dụng.
                    + Chức năng biểu diễn, vận chuyển thông tin để trình diễn (view) và xử lý (control)
                    + Chứa tất cả các nghiệp vụ logic, đối tượng mô tả dữ liệu, ...

                 > view: hiển thị cho người dùng.(ví dụ đây là giao diện java Swing)
                    + Tương tác với người sử dụng.
                    + Show kết quả từ tầng Controller.
                    + Thu nhận các hoạt động, request của người sử dụng và chuyển cho tầng Controller xử lý.
                    + Hiểu một cách đơn giản, View là hệ thống các frame, cửa sổ của ứng dụng; các trang giao diện web: html, jsp; Các bảng, mẫu biểu, báo cáo.

                 > Controle: có chức năng điều khiển tương tác giữa Model và view (các sự kiện.. nút chức năng hiển thị trên giao diện)
                    + Định nghĩa các hành vi, hoạt động, xử lý của hệ thống.
                    + Đối chiếu hành động của người sử dụng từ View. Đồng thời tương tác Model để gọi View và hiển thị thông tin tương
                    ứng cho người dùng.
                    
                 > use: là người dùng tương tác
